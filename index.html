What is KBAI and its role in AI?
🔥 مهم جداًEssayMCQ
ممكن ييجي: "Explain KBAI" · "What makes KBAI different?"
KBAI بيستخدم explicit knowledge (rules, logic, ontologies) عشان يعمل reasoning — مش بيتعلم من data زي ML.
KBAI = Knowledge Base + Inference Engine + Explanation Facility
كلمات لازم تكتبها:
Symbolic AIExplicit KnowledgeInference EngineExplainable ReasoningMYCINKBAI ⊂ AI
KBAI: transparent, rule-based
مثال: MYCIN بيشخص bacterial infections بـ 600 rule ويشرح
Data-driven AI: black-box, neural network
مثال: Deep Learning — مش ممكن يشرح قراره
النقطة الأهم: KBAI بيقدر يشرح القرار لأنه بيستخدم rules واضحة مش أرقام إحصائية.

أهم Definitions

Knowledge Base (KB)
🔥 مهم جداًMCQ
مستودع منظم من facts + rules + relationships
مثال: Medical KB = أمراض + أعراض + علاجات
Inference Engine
🔥 مهم جداًMCQ
المحرك اللي بيستنتج معرفة جديدة من الـ KB
مثال: backward chaining عشان يشخص السكري
Expert System
🔥 مهم جداًMCQ
نظام KBAI = KB + Inference Engine
المثال الدائم: MYCIN — bacterial infections
Forward Chaining
🔥 مهم جداًMCQ
Data-driven — بيبدأ من facts يستنتج conclusions
Facts → Rules → Conclusion
Backward Chaining
🔥 مهم جداًMCQ
Goal-driven — بيبدأ من goal يتحقق من facts
Goal → Check Rules → Check Facts
Semantic Network
🔥 مهم جداًMCQ
graph: مفاهيم كـ nodes وعلاقاتهم كـ edges
Bird → is-a → Animal | Bird → has → Wings
Ontology
🔥 مهم جداً
تمثيل رسمي: concepts + relationships + constraints
⚠️ Ontology ≠ Taxonomy (taxonomy = hierarchy بس)
Production Rule
🔥 مهم جداًMCQ
جملة IF-THEN بتمثل procedural knowledge
IF Temperature > 30°C THEN TurnOn(AC)
Rational Agent
🔥 مهم جداً
بيختار الـ action اللي يُعظّم performance بناءً على المعلومات
⚠️ Rational ≠ Always Correct!
Frame
⚡ مهمMCQ
هيكل بيمثل object بـ slots + values
Car: {"{type: vehicle, wheels: 4}"}
Model-Based Agent
⚡ مهمMCQ
عنده internal model عن العالم بيستخدمه في القرارات
مثال: Self-driving car عنده model للطريق والعوائق
Utility-Based Agent
⚡ مهمMCQ
بيختار actions باستخدام utility function لتعظيم الفائدة
مثال: Smart thermostat يوازن الراحة وتوفير الطاقة
Non-Monotonic Reasoning
⚡ مهم
reasoning بيُعيد تقييمه لو جت معلومة جديدة تناقض الاستنتاجات
Tweety طيرة → يطير | اكتشفنا إنه بطريق → مش بيطير
Explanation Facility
مهم
بيشرح ليه النظام وصل لقرار معين
MYCIN: "لأن الحرارة + WBC + symptom X"
CSP
مهمMCQ
تعيين قيم لـ variables مع احترام constraints
Variables + Domains + Constraints
مثال: جدول محاضرات — مش قاعتين في نفس الوقت
Declarative Knowledge
مهم
معرفة بالحقائق — "knowing THAT"
"كل الطيور ليها أجنحة" — يُخزن في frames/ontologies
Procedural Knowledge
مهم
معرفة بكيفية التنفيذ — "knowing HOW"
خطوات حل معادلة — يُخزن في production rules
Explicit Knowledge
عارفه
واضحة ممكن تُخزن وتُشارك
قوانين قانونية، بروتوكولات طبية
Tacit Knowledge
عارفه
ضمنية وتجريبية — صعب ترميزها
طباخ بيوازن النكهات من غير وصفة

نوع: Goal-driven

مناسب لـ: diagnosis, proving

تركات الدكتور

⚠️ Rational ≠ Correct
Rational = أحسن قرار بناءً على المعلومات — مش لازم صح دايماً
⚠️ KBAI ≠ AI (General)
KBAI = subset من AI — Deep Learning مش KBAI
⚠️ Ontology ≠ Taxonomy
Taxonomy = hierarchy بس | Ontology = hierarchy + relationships + constraints
⚠️ KB ≠ KBS ≠ KBAI
KB = storage | KBS = KB + reasoning | KBAI = KBS + AI + explainability
⚠️ Forward ≠ Backward
Forward = بدأ من data | Backward = بدأ من goal
⚠️ MYCIN في MCQ
دايماً = Expert System / KBS — مش self-driving car 😅
⚠️ Non-Monotonic = قابل للتعديل
Tweety طيرة → يطير | اكتشفنا إنه بطريق → مش بيطير
⚠️ "explain reasoning" في السؤال
فوراً فكر في Explanation Facility
⚠️ CSP مش نوع Agent
CSP = نوع مشكلة — مش نوع agent
⚠️ Simple Reflex = بدون memory
بيرد على stimulus حالي فقط — عكسه Model-Based
